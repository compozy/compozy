# BUG-001: UI token package omitted Disket Mono font faces

## Status

Fixed

## Severity

Medium

## Discovered By

`make verify` during `frontend:test`.

## Environment

- Workspace: `/Users/pedronauck/Dev/compozy/looper`
- Command: `make verify`
- Test: `packages/ui/tests/tokens.test.ts`

## Expected

The shared UI token package ships the mockup font faces used by the daemon web UI design system, including `Nippo` and `Disket Mono`.

## Actual

`packages/ui/src/tokens.css` declared `Nippo` but omitted both packaged `Disket Mono` faces, while the assets were still present under `packages/ui/src/assets/fonts/`.

## Root Cause

The production token CSS drifted from the packaged design assets. This caused the token contract test to fail and would have prevented the built web bundle from including the Disket Mono font files.

## Fix

Restored `@font-face` declarations for:

- `Disket-Mono-Regular.ttf` at weight `400`
- `Disket-Mono-Bold.ttf` at weight `700`

## Verification

- `bun run --cwd packages/ui test` passed: `5` test files, `22` tests.
- `make verify` passed end-to-end.
- The production web build output includes `Disket-Mono-Regular-*.ttf` and `Disket-Mono-Bold-*.ttf`.
