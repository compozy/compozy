## Context

The former raw hooks inverted the intended quality ladder: pre-commit ran a slow
whole-repository typecheck while pre-push ran only the build. The real CI checks
are independent, but a sequential local gate sums their durations and encourages
`--no-verify`. Typecheck and build also shared Paraglide code generation, creating a
write race if they were naively parallelized.

## Decision

Use Lefthook as the only Git-hook owner.

- Pre-commit remains seconds-fast: Biome checks staged supported files and the
  optional GitGuardian helper runs when armed; no typecheck runs here.
- Pre-push runs Paraglide codegen exactly once, then executes lint and documentation
  guards, `tsc --noEmit`, the full workspace test suite, and production build plus
  SSR smoke in parallel. Any check failure blocks the push.
- Git's native `--no-verify` escape hatch remains documented.
- Process cleanup, interruption, bypass, and cross-workspace failure behavior are
  tested through real temporary Git repositories rather than sentinel-only mocks.

## Alternatives

- Keep raw shell hooks. Rejected because reliable staged-file handling,
  parallelism, ordering, and cleanup would become custom shell infrastructure.
- Use Husky plus lint-staged. Rejected because it adds more Node-oriented machinery
  than the single Lefthook binary in this Bun repository.
- Run only affected tests. Rejected because test-only changes and dynamic imports
  can escape selection, breaking local-to-CI parity.

## Consequences

- Pre-push wall time is bounded mostly by the slowest check instead of their sum.
- The full suite becomes the default local predictor of CI rather than a heuristic.
- Lefthook and its ordering model are project infrastructure that future features
  must extend instead of installing competing hooks.
- Parallel output and long-running integration cleanup require dedicated harness
  code and regression coverage.

## Reconciliation

The hook ownership, codegen barrier, parallel full-suite checks, and documented
bypass were implemented as designed. Review remediation removed the legacy
`core.hooksPath`, exercised real failure modes, serialized the integration project
where needed, and hardened child-process teardown.

**[DEVIATION]** The settled pre-push gate is broader than the original four-lane
plan. Documentation guards run with lint and the Bun SSR smoke runs after the
production build so local pre-push matches the checks that CI actually enforces.
Resolved reviews-010/issue_002 and reviews-011/issue_001 established this expanded
parity contract. The non-blocking Graphify refresh remains outside the checks and
is captured separately in AD-014.
