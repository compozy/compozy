## Context

Agents use Graphify's local knowledge graph to navigate the codebase, but the graph
is derived state. Committing it would create large JSON churn and require a merge
driver; Graphify's own installable hook would also compete with Lefthook and burden
the seconds-fast pre-commit. A purely manual refresh, however, silently becomes
stale.

## Decision

Treat `graphify-out/` as a per-clone, gitignored, regenerable index.

- Lefthook remains the only hook owner; do not run `graphify hook install` and do
  not configure a graph merge driver.
- After all blocking pre-push checks pass, a final piped job runs
  `graphify update . || true`.
- The refresh is non-blocking and no-ops until a local graph exists, so Graphify
  availability can never change push correctness or CI parity.
- Only the AST-only update belongs in the hook. The LLM/API-backed initial
  `graphify .` build remains an explicit `make graph` operation.

## Alternatives

- Commit the graph and use Graphify's native hook/merge-driver model. Rejected due
  to derivative-file churn, merge maintenance, and competing hook ownership.
- Refresh on pre-commit or every file change. Rejected because it taxes the fast
  loop and introduces background-process management.
- Refresh only on demand. Rejected because agents would routinely query stale
  structure.

## Consequences

- A green pushed batch refreshes local navigation data at zero API cost.
- Fresh clones and hosted CI have no graph until an explicit initial build.
- The graph may include uncommitted working-tree state, which is acceptable because
  it is a local view of the tree the agent is actually editing.
- Future hook work must preserve Graphify's last, non-blocking position.

## Reconciliation

Implemented as designed. `graphify-out/` is ignored, `make graph` owns the initial
build, and Lefthook's final graphify job is outside the blocking parallel checks and
uses `|| true`. IT-028 through IT-030 cover absence/failure, refresh-on-green, and
ordering. Resolved reviews-005/issue_004 replaced a vacuous string-position test
with evidence that the graphify job is structurally last.

The current capture itself successfully queried the local graph before inspecting
the codebase, confirming that the per-clone consumption path is live without making
the graph part of version control.
