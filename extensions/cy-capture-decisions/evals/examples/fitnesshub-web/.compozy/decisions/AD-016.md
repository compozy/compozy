## Context

The full pre-push matrix delayed every publication of AI-assisted work even though
`make full-verify` already exists as the stronger local merge-evidence gate. The
automatic Graphify refresh added more synchronous push work despite being derived,
non-gating local state.

## Decision

Keep only staged-file Biome and the optional GitGuardian helper in Lefthook's
pre-commit hook. Do not configure a pre-push hook.

Use `make verify` only when mid-development whole-repository evidence is useful.
Before merge, fetch the target branch and run `make full-verify`; record the exact
verified commit SHA in the PR, and rerun the gate after any later commit.

Run `graphify update .` explicitly after code changes. It remains local,
gitignored, regenerable state and never blocks commit, push, or merge verification.

## Consequences

- Commits retain fast staged feedback and optional secret protection.
- Pushes no longer run codegen, lint, documentation guards, typecheck, tests,
  build, SSR smoke, or Graphify maintenance.
- Merge confidence depends on the maintainer running `make full-verify` against the
  exact SHA being approved.
- A local pass remains evidence, not a GitHub status; hosted checks and
  GitGuardian's hosted scan retain the limitations recorded by AD-015.

## Reconciliation

AD-011's automatic full-suite pre-push gate and AD-014's automatic Graphify refresh
are retired. Lefthook remains the sole hook manager for the one retained
pre-commit hook. The obsolete pre-push integration suite is removed; shipped
configuration and fresh-clone tests enforce the smaller contract.
