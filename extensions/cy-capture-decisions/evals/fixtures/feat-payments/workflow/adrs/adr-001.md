# ADR-001: Idempotency keys on payment writes

- Status: Accepted
- Date: 2026-07-11

## Context

Clients retry payment requests on timeouts. Any feature issuing writes to the payment ledger must avoid
double-charging.

## Decision

Require an `Idempotency-Key` header on every payment write; dedupe on it server-side for 24h.

## Alternatives Considered

- Client-side dedupe only (rejected: unreliable across retries and clients).

## Consequences

- Cross-feature-durable: every writer to the ledger must honor the key convention.
