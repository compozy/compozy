# ADR-001: Server-side sessions with revocation

- Status: Accepted
- Date: 2026-07-11

## Context

A security review required immediate session revocation on credential compromise. Stateless JWTs
(feat-sessions, AD-001) cannot revoke before expiry. This decision reverses the prior one.

## Decision

Move to server-side sessions with a revocation list; drop stateless JWTs as the session mechanism.

## Alternatives Considered

- Short-lived JWTs + refresh (rejected: still a revocation window).

## Consequences

- Cross-feature-durable and conflicts with the active AD-001 (stateless JWT sessions) → SUPERSEDE.
