# ADR-001: Inverted-index search backend

- Status: Accepted
- Date: 2026-07-11

## Context

Search must scale to full-text queries across large corpora; the backend choice constrains every future
search-consuming feature.

## Decision

Use an inverted index (per-term posting lists) as the search backend.

## Alternatives Considered

- Trigram scan (rejected: does not scale to the corpus size).

## Consequences

- Cross-feature-durable: ranking, faceting, and typeahead all build on this backend.
