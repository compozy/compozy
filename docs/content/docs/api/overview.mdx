## Getting Started

<Steps numbered size="sm" className="mt-12">
<Step title="Configure Base URL" description="Set up your API client with the correct Compozy endpoint. All API endpoints are available at:">

```
http://<your-compozy-instance>/api/v0
```

</Step>

<Step title="Authentication Setup" description="Configure secure access for API requests">

Most endpoints require authentication. Include your API key in requests:

```bash
Authorization: Bearer your_api_key_here
```

</Step>

<Step title="Make Your First Request" description="Test the API with a simple workflow execution">

<Tabs items={["cURL", "JavaScript", "Python", "HTTP Client"]} className="mt-6">
<Tab>

```bash
# Execute a workflow
curl -X POST http://localhost:3001/api/v0/workflows/my-workflow/executions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer your_api_key" \
  -d '{
    "input": {
      "message": "Hello, world!"
    }
  }'
```

</Tab>

<Tab>

```javascript
const response = await fetch('http://localhost:3001/api/v0/workflows/my-workflow/executions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your_api_key'
  },
  body: JSON.stringify({
    input: {
      message: 'Hello, world!'
    }
  })
});

const result = await response.json();
console.log('Execution ID:', result.data.exec_id);
```

</Tab>

<Tab>

```python
import requests

response = requests.post(
    'http://localhost:3001/api/v0/workflows/my-workflow/executions',
    headers={
        'Content-Type': 'application/json',
        'Authorization': 'Bearer your_api_key'
    },
    json={
        'input': {
            'message': 'Hello, world!'
        }
    }
)

result = response.json()
print(f"Execution ID: {result['data']['exec_id']}")
```

</Tab>

<Step title="Monitor Execution" description="Track workflow progress and retrieve results">

Check execution status and retrieve outputs:

```bash
# Get execution status
curl -X GET http://localhost:3001/api/v0/executions/workflows/{execution_id} \
  -H "Authorization: Bearer your_api_key"

# Get execution outputs (when completed)
curl -X GET http://localhost:3001/api/v0/executions/workflows/{execution_id}/outputs \
  -H "Authorization: Bearer your_api_key"
```

## Response Format Standards

<Tabs items={["Success Response", "Error Response", "Execution Status", "Streaming Response"]}>
<Tab>

**Standard Success Response:**
```json
{
  "data": {
    "exec_id": "exec_01234567-89ab-cdef-0123-456789abcdef",
    "workflow_id": "my-workflow",
    "status": "running",
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T10:30:00Z"
  },
  "message": "Workflow execution started successfully"
}
```

</Tab>

<Tab>

**Error Response Structure:**
```json
{
  "error": "VALIDATION_FAILED",
  "details": "Input validation failed: field 'city' is required",
  "timestamp": "2024-01-15T10:30:00Z",
  "request_id": "req_12345"
}
```

**Common Error Codes:**
- `VALIDATION_FAILED` - Input validation errors
- `WORKFLOW_NOT_FOUND` - Requested workflow doesn't exist
- `EXECUTION_FAILED` - Workflow execution encountered an error
- `AUTHENTICATION_REQUIRED` - Missing or invalid API key

</Tab>

<Tab>

**Execution Status Response:**
```json
{
  "data": {
    "exec_id": "exec_01234567-89ab-cdef-0123-456789abcdef",
    "workflow_id": "my-workflow",
    "status": "completed",
    "progress": {
      "completed_tasks": 3,
      "total_tasks": 3,
      "percentage": 100
    },
    "outputs": {
      "result": "Success",
      "processed_items": 42
    },
    "duration": "00:02:34",
    "created_at": "2024-01-15T10:30:00Z",
    "completed_at": "2024-01-15T10:32:34Z"
  }
}
```

**Status Values:**
- `pending` - Execution queued
- `running` - Currently executing
- `completed` - Successfully finished
- `failed` - Execution failed
- `cancelled` - Manually cancelled

</Tab>

<Tab>

**Streaming Response (Server-Sent Events):**
```
event: execution.started
data: {"exec_id": "exec_123", "workflow_id": "my-workflow"}

event: task.completed
data: {"task_id": "task_1", "status": "completed", "output": "..."}

event: execution.completed
data: {"exec_id": "exec_123", "status": "completed", "outputs": "..."}
```

</Tab>
</Tabs>
