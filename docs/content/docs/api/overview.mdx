## Getting Started

http://<your-compozy-instance>/api/v0

</Step>

<Step title="Authentication Setup" description="Configure secure access for API requests">

Most endpoints require authentication. Include your API key in requests:

```bash
Authorization: Bearer {{ COMPOZY_API_KEY }}
```

</Step>

<Callout type="warning">
Never commit real API keys. Replace `{{ COMPOZY_API_KEY }}` with a secret loaded from your environment (for example, `export COMPOZY_API_KEY=...`).
</Callout>

## Streaming Executions

Server-Sent Events (SSE) endpoints provide live updates for workflows, agents, and tasks. Streams emit ordered events with `id`, `event`, `ts`, and `data` fields and support automatic reconnection via the `Last-Event-ID` header.

- `GET /executions/workflows/{exec_id}/stream` &mdash; durable JSON events with resume support.
- `GET /executions/agents/{exec_id}/stream` &mdash; structured JSON when the agent declares an output schema, otherwise progressive `llm_chunk` text lines.
- `GET /executions/tasks/{exec_id}/stream` &mdash; mirrors the agent behavior for task executions.

Use query parameters to tailor delivery:

- `poll_ms` sets the polling cadence for durable stores (default `500`, min `250`, max `2000` milliseconds).
- `events` lets clients provide a comma-separated allowlist (for example `workflow_status,tool_call,llm_chunk`).
- `Last-Event-ID` resumes workflow and structured streams from the last confirmed event.

<Tabs items={["JavaScript", "curl"]}>
<Tab>

```javascript
const execId = "01JZP6D1XSYQ1JR2QH9AZ2T0TB";
const stream = new EventSource(
  `/api/v0/executions/workflows/${execId}/stream?poll_ms=500`,
  { withCredentials: false }
);

stream.addEventListener("workflow_status", event => {
  const payload = JSON.parse(event.data);
  renderStatus(payload);
});

stream.addEventListener("tool_call", event => {
  console.info("Tool call:", JSON.parse(event.data));
});

stream.addEventListener("llm_chunk", event => {
  appendConsoleLine(event.data);
});

stream.addEventListener("complete", () => {
  stream.close();
});
```

</Tab>

<Tab>

```bash
curl -N "http://localhost:5001/api/v0/executions/agents/${AGENT_EXEC_ID}/stream?events=agent_status,llm_chunk" \
  -H "Authorization: Bearer ${COMPOZY_API_KEY}" \
  -H "Accept: text/event-stream" \
  -H "Last-Event-ID: 42"
```

</Tab>
</Tabs>

For event field definitions and payload formats, see the [Execution Stream Events schema](/docs/schema/execution-stream-events).

## Response Format Standards

<Tabs items={["Success Response", "Error Response", "Execution Status", "Streaming Response"]}>
<Tab>

**Standard Success Response:**
```json
{
  "data": {
    "exec_id": "exec_01234567-89ab-cdef-0123-456789abcdef",
    "workflow_id": "my-workflow",
    "status": "running",
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T10:30:00Z"
  },
  "message": "Workflow execution started successfully"
}
```

</Tab>

<Tab>

**Error Response Structure:**
```json
{
  "error": "VALIDATION_FAILED",
  "details": "Input validation failed: field 'city' is required",
  "timestamp": "2024-01-15T10:30:00Z",
  "request_id": "req_12345"
}
```

**Common Error Codes:**
- `VALIDATION_FAILED` - Input validation errors
- `WORKFLOW_NOT_FOUND` - Requested workflow doesn't exist
- `EXECUTION_FAILED` - Workflow execution encountered an error
- `AUTHENTICATION_REQUIRED` - Missing or invalid API key

</Tab>

<Tab>

**Execution Status Response:**
```json
{
  "data": {
    "exec_id": "exec_01234567-89ab-cdef-0123-456789abcdef",
    "workflow_id": "my-workflow",
    "status": "completed",
    "progress": {
      "completed_tasks": 3,
      "total_tasks": 3,
      "percentage": 100
    },
    "outputs": {
      "result": "Success",
      "processed_items": 42
    },
    "duration": "00:02:34",
    "created_at": "2024-01-15T10:30:00Z",
    "completed_at": "2024-01-15T10:32:34Z"
  }
}
```

**Status Values:**
- `pending` - Execution queued
- `running` - Currently executing
- `completed` - Successfully finished
- `failed` - Execution failed
- `cancelled` - Manually cancelled

</Tab>

<Tab>

**Streaming Response (Server-Sent Events):**
```
event: execution.started
data: {"exec_id": "exec_123", "workflow_id": "my-workflow"}

event: task.completed
data: {"task_id": "task_1", "status": "completed", "output": "..."}

event: execution.completed
data: {"exec_id": "exec_123", "status": "completed", "outputs": "..."}
```

</Tab>
</Tabs>
