### Authentication Methods

<Tabs items={["JWT Authentication", "OAuth 2.0", "API Keys", "Role-Based Access"]}>

<Tab>
**JWT Authentication** provides secure token-based authentication:

```bash
# Authenticate and get JWT token
curl -X POST "http://localhost:8080/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "your-secure-password"
  }'

# Response
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 3600,
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user_info": {
    "id": "admin",
    "roles": ["admin", "operator"],
    "permissions": ["server:read", "server:write", "system:admin"]
  }
}
```

**Using JWT token for API requests:**

```bash
# Use JWT token in Authorization header
curl -X GET "http://localhost:8080/api/v1/servers" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

</Tab>

<Tab>
**OAuth 2.0** integration with enterprise identity providers:

```yaml
# config/auth.yaml
oauth:
  provider: "okta"  # okta, azure, google, keycloak
  client_id: "your-client-id"
  client_secret: "your-client-secret"
  redirect_uri: "http://localhost:8080/auth/callback"
  scope: "openid profile email"
  
  # Provider-specific configuration
  discovery_url: "https://your-domain.okta.com/.well-known/openid_configuration"
  
  # Claims mapping
  claims:
    user_id: "sub"
    username: "preferred_username"
    email: "email"
    roles: "groups"
```

**OAuth flow example:**

```bash
# Step 1: Redirect to OAuth provider
GET /auth/oauth/authorize?provider=okta

# Step 2: Handle callback with authorization code
GET /auth/oauth/callback?code=authorization_code&state=random_state

# Step 3: Exchange code for access token
POST /auth/oauth/token
{
  "access_token": "oauth_access_token",
  "refresh_token": "oauth_refresh_token",
  "user_info": {
    "id": "user123",
    "email": "user@company.com",
    "roles": ["mcp-operator"]
  }
}
```

</Tab>

<Tab>
**API Keys** for service-to-service authentication:

```bash
# Generate API key
curl -X POST "http://localhost:8080/api/v1/auth/api-keys" \
  -H "Authorization: Bearer jwt_token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "monitoring-service",
    "description": "API key for monitoring service",
    "permissions": ["metrics:read", "health:read"],
    "expires_at": "2024-12-31T23:59:59Z"
  }'

# Response
{
  "id": "ak_1234567890",
  "name": "monitoring-service",
  "key": "mcp_ak_1234567890abcdef...",
  "permissions": ["metrics:read", "health:read"],
  "created_at": "2024-01-01T00:00:00Z",
  "expires_at": "2024-12-31T23:59:59Z"
}
```

**Using API key:**

```bash
# Use API key in X-API-Key header
curl -X GET "http://localhost:8080/api/v1/metrics" \
  -H "X-API-Key: mcp_ak_1234567890abcdef..."
```

</Tab>

### WebSocket Connection

```javascript
// Connect to WebSocket API
const ws = new WebSocket('ws://localhost:8080/api/v1/ws');

// Authentication
ws.onopen = function() {
  ws.send(JSON.stringify({
    type: 'auth',
    token: 'your-jwt-token'
  }));
};

// Subscribe to events
ws.send(JSON.stringify({
  type: 'subscribe',
  channels: ['server-health', 'metrics', 'tool-executions']
}));

// Handle incoming messages
ws.onmessage = function(event) {
  const data = JSON.parse(event.data);
  console.log('Received:', data);
};
```

## SDK and Client Libraries

<Callout type="info" icon={Package}>
  **Developer Tools**: Official SDKs and client libraries for seamless integration with the Admin API.
</Callout>

### JavaScript/TypeScript SDK

```bash
npm install @compozy/admin-api-sdk
```

**Basic Usage:**

```typescript
import { AdminAPIClient } from '@compozy/admin-api-sdk';

const client = new AdminAPIClient({
  baseURL: 'http://localhost:8080',
  token: 'your-jwt-token'
});

// Server management
const servers = await client.servers.list();
const server = await client.servers.create({
  id: 'new-server',
  name: 'New Server',
  type: 'stdio',
  configuration: {
    command: '/usr/local/bin/mcp-server',
    args: ['--config', '/etc/mcp/config.json']
  }
});

// Tool execution
const result = await client.tools.execute('filesystem-server', 'list_files', {
  path: '/var/lib/workspace'
});

// Metrics
const metrics = await client.metrics.getSystemMetrics('1h');
```
