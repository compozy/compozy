## What Are Tools?

Tools are the building blocks that extend your AI agents' capabilities beyond text generation. They're [TypeScript functions](/docs/core/tools/typescript-development) that agents can call during workflow execution, running in a secure, sandboxed [Bun runtime environment](/docs/core/tools/runtime-environment).

<Callout type="info">
Tools enable agents to interact with external systems, process data, and perform complex operations that require deterministic logic and type safety.
</Callout>

## Why Use Tools?

When building AI-powered applications, you often need agents to perform specific actions like fetching data from APIs, processing files, or interacting with databases. Tools provide a structured, type-safe interface for these operations while maintaining security and performance.

## How Tools Work

<Steps>
  <Step title="Agent Request" icon="User">
    During workflow execution, an agent decides to use a tool based on the task requirements
  </Step>
  <Step title="Input Validation" icon="CheckCircle">
    The tool's input is validated against its JSON Schema definition
  </Step>
  <Step title="Isolated Execution" icon="Shield">
    The tool runs in a sandboxed Bun process with configured permissions
  </Step>
  <Step title="Output Processing" icon="ArrowUpCircle">
    Results are validated and returned to the agent for further processing
  </Step>
</Steps>

## Core Features

### Schema-First Design

Every tool defines its interface using JSON Schema, providing:

<List>
  <ListItem title="Type Safety" icon="ShieldCheck">Catch errors before runtime with validated inputs/outputs</ListItem>
  <ListItem title="Self-Documentation" icon="BookOpen">Clear parameter descriptions for agents and developers</ListItem>
  <ListItem title="LLM Integration" icon="FunctionSquare">Seamless function calling with structured data</ListItem>
</List>

```yaml
input:
  type: object
  properties:
    query:
      type: string
      description: "Search query to execute"
    limit:
      type: integer
      default: 10
      minimum: 1
      maximum: 100
  required: [query]
```

### Secure Execution Environment

Tools run in isolated [Bun processes](/docs/core/tools/runtime-environment) with granular security controls:

<List>
  <ListItem title="Permission System" icon="Lock">Control file, network, and environment access</ListItem>
  <ListItem title="Resource Limits" icon="Gauge">Set memory, CPU, and timeout constraints</ListItem>
  <ListItem title="Environment Isolation" icon="Shield">Secure credential management with filtered variables</ListItem>
  <ListItem title="Output Validation" icon="CheckCircle">Prevent memory exhaustion with size limits</ListItem>
</List>

```yaml
runtime:
  permissions:
    - --allow-read=./data
    - --allow-net=api.example.com
  limits:
    memory: "256MB"
    timeout: "30s"
```

Built-in optimizations ensure tools run efficiently:

## Basic Tool Example

<Tabs items={["TypeScript Implementation", "YAML Configuration", "Workflow Usage"]}>
  <Tab value="TypeScript Implementation">
    ```typescript
    // tools/data-processor.ts
    interface Input {
      data: unknown[];
      operation: 'filter' | 'map' | 'reduce';
      expression: string;
    }

    interface Output {
      result: unknown;
      processed: number;
      duration: number;
    }

    export async function dataProcessor(input: Input): Promise<Output> {
      const start = Date.now();

      // Validate and process data
      if (!Array.isArray(input.data)) {
        throw new Error('Input data must be an array');
      }

      let result: unknown;
      switch (input.operation) {
        case 'filter':
          result = input.data.filter(item =>
            evaluate(item, input.expression)
          );
          break;
        case 'map':
          result = input.data.map(item =>
            transform(item, input.expression)
          );
          break;
        case 'reduce':
          result = input.data.reduce((acc, item) =>
            aggregate(acc, item, input.expression),
            {}
          );
          break;
      }

      return {
        result,
        processed: input.data.length,
        duration: Date.now() - start
      };
    }
    ```
  </Tab>

  <Tab value="YAML Configuration">
    ```yaml
    resource: tool
    id: data-processor
    description: Process arrays with filter, map, or reduce operations

    input:
      type: object
      properties:
        data:
          type: array
          description: Array of items to process
        operation:
          type: string
          enum: [filter, map, reduce]
          description: Operation to perform
        expression:
          type: string
          description: Expression or function to apply
      required: [data, operation, expression]

    output:
      type: object
      properties:
        result:
          description: Processed data result
        processed:
          type: integer
          description: Number of items processed
        duration:
          type: integer
          description: Processing time in milliseconds

    runtime:
      timeout: "60s"
      permissions:
        - --allow-read
    ```
  </Tab>

  <Tab value="Workflow Usage">
    ```yaml
    tasks:
      - id: process_customer_data
        type: basic
        agent: data-analyst
        action: analyze
        $use: tool(local::tools.data-processor)
        with:
          data: "{{ .inputs.customers }}"
          operation: filter
          expression: "age > 18 AND country = 'US'"
        outputs:
          filtered_customers: "{{ .output.result }}"
          count: "{{ .output.processed }}"
    ```
  </Tab>
</Tabs>
