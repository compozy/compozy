## Quick Start

<Steps numbered>
  <Step title="Initialize Project" description="Create your project directory and configuration file">
    Create a new directory for your project and initialize the basic configuration:

    ```bash
    mkdir my-ai-project
    cd my-ai-project
    ```

    Create a minimal `compozy.yaml` file:

    ```yaml
    name: my-ai-project
    version: 1.0.0
    description: My AI project

    workflows:
      - source: ./workflow.yaml

    models:
      - provider: openai
        model: gpt-4
        api_key: "{{ .env.OPENAI_API_KEY }}"
    ```

    <Callout type="info">
      For detailed provider configuration including multiple models, authentication methods, and advanced settings, see the [Provider Configuration](/docs/core/configuration/providers) guide.
    </Callout>
  </Step>

  <Step title="Environment Variables" description="Set up authentication and secrets">
    Create a `.env` file with your API keys and configuration:

    ```bash
    # .env
    OPENAI_API_KEY=sk-...
    ANTHROPIC_API_KEY=sk-ant-...
    ```

    <Callout type="warning">
      Never commit API keys to version control. Use environment variables or secure secret management.
    </Callout>
  </Step>

  <Step title="Create Workflow" description="Define your first workflow">
    Create a `workflow.yaml` file with your AI workflow:

    ```yaml
    name: basic-workflow
    version: 1.0.0
    description: Basic AI workflow

    agents:
      - id: assistant
        instructions: You are a helpful AI assistant.
        model: gpt-4

    tasks:
      - id: process-input
        type: basic
        agent: { id: assistant }
        with:
          input: "{{ .workflow.input.message }}"
    ```

    <Callout type="info">
      Learn more about workflow configuration, task types, and advanced orchestration patterns in the [Workflow Configuration](/docs/core/configuration/workflows) guide.
    </Callout>
  </Step>

  <Step title="Test Configuration" description="Validate your setup">
    Test your configuration with the CLI:

    ```bash
    compozy dev
    ```

    Your project is now ready for development!

    <Callout type="info">
      For CLI configuration options, development commands, and debugging tools, see the [CLI Configuration](/docs/core/configuration/cli) guide.
    </Callout>
  </Step>
</Steps>

## Configuration Examples

<Tabs items={["Minimal", "Production", "Development"]}>
  <Tab>
    **Minimal Configuration**

    Perfect for getting started quickly:

    ```yaml
    name: simple-ai-app
    version: 1.0.0
    description: Simple AI application

    workflows:
      - source: ./workflow.yaml

    models:
      - provider: openai
        model: gpt-4
        api_key: "{{ .env.OPENAI_API_KEY }}"
    ```
  </Tab>

  <Tab>
    **Production Configuration**

    Comprehensive setup for production environments:

    ```yaml
    name: enterprise-ai-system
    version: 2.1.0
    description: Multi-agent system for enterprise automation

    author:
      name: AI Team
      email: ai@company.com
      organization: ACME Corp

    workflows:
      - source: ./workflows/customer-support.yaml
      - source: ./workflows/data-pipeline.yaml

    models:
      - provider: openai
        model: gpt-4-turbo
        api_key: "{{ .env.OPENAI_API_KEY }}"
        temperature: 0.7
        max_tokens: 4000
      - provider: anthropic
        model: claude-3-opus
        api_key: "{{ .env.ANTHROPIC_API_KEY }}"
        # See Provider Configuration docs for detailed settings

    runtime:
      type: bun
      entrypoint: ./tools/index.ts
      permissions:
        - --allow-read=/data
        - --allow-net=api.company.com
        - --allow-env=API_KEY,DATABASE_URL
        # See Runtime Configuration docs for security settings

    cache:
      url: redis://localhost:6379/0
      pool_size: 10
      key_prefix: "compozy:"

    monitoring:
      enabled: true
      metrics:
        provider: prometheus
        endpoint: /metrics
      logging:
        level: info
        format: json

    config:
      max_string_length: 52428800  # 50MB
      async_token_counter_workers: 20
      dispatcher_heartbeat_interval: 30
    ```
  </Tab>

  <Tab>
    **Development Configuration**

    Optimized for development with hot-reload and debugging:

    ```yaml
    name: dev-ai-project
    version: 0.1.0
    description: Development AI project

    workflows:
      - source: ./workflow.yaml

    models:
      - provider: openai
        model: gpt-4
        api_key: "{{ .env.OPENAI_API_KEY }}"
      - provider: ollama
        model: llama2:13b
        api_url: http://localhost:11434

    runtime:
      type: bun
      entrypoint: ./tools.ts
      permissions:
        - --allow-read
        - --allow-net
        - --allow-env

    autoload:
      enabled: true
      strict: true
      watch_interval: 2s
      include:
        - "agents/**/*.yaml"
        - "workflows/**/*.yaml"
        - "memory/**/*.yaml"
      exclude:
        - "**/*.tmp"
        - "**/*~"
        # See AutoLoad Configuration docs for pattern options

    config:
      max_string_length: 10485760  # 10MB
      async_token_counter_workers: 5
    ```
  </Tab>
</Tabs>

        Use clear, descriptive names for projects, workflows, and agents

      </ListItem>
      <ListItem title="Documentation" icon={FileText}>
