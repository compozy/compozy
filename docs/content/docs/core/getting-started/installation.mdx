## Install Compozy CLI

Choose your preferred installation method. We support various package managers, pre-built binaries, and Docker for easy installation.

<Tabs items={["Homebrew", "NPM", "Go Install", "Docker", "Linux Packages", "Arch Linux (AUR)", "Nix", "Pre-built Binary"]}>

<Tab>
**For macOS and Linux users:**
```bash
# Install via Homebrew
brew tap compozy/homebrew-compozy
brew install compozy
```
</Tab>

<Tab>
**For Node.js developers:**
```bash
# Install via NPM
npm install -g @compozy/cli

# Or with Yarn
yarn global add @compozy/cli

# Or with pnpm
pnpm add -g @compozy/cli
```
</Tab>

<Tab>
**For developers with a Go environment:**
```bash
# Install via Go
go install github.com/compozy/compozy@latest
```
</Tab>

<Tab>
**For containerized environments:**
```bash
# Pull the latest Docker image
docker pull ghcr.io/compozy/compozy:latest

# Run Compozy using Docker
docker run --rm -it ghcr.io/compozy/compozy:latest --version
```
</Tab>

<Tab>
Download the appropriate `.deb`, `.rpm`, or `.apk` package for your distribution from the [GitHub Releases](https://github.com/compozy/compozy/releases) page and install it using your system's package manager.

**For Debian/Ubuntu:**
```bash
# Replace with the downloaded file name
sudo dpkg -i compozy_*.deb
```

**For Fedora/CentOS:**
```bash
# Replace with the downloaded file name
sudo rpm -i compozy_*.rpm
```

**For Alpine Linux:**
```bash
# Replace with the downloaded file name
sudo apk add --allow-untrusted compozy_*.apk
```
</Tab>

<Tab>
Install from the Arch User Repository (AUR) using an AUR helper such as `yay`.

**Install from source:**
```bash
yay -S compozy
```

**Install pre-built binary:**
```bash
yay -S compozy-bin
```
</Tab>

<Tab>
**For Nix users:**
```bash
# Install via Nix
nix-env -iA nixpkgs.compozy

# Or with Home Manager
home.packages = [ pkgs.compozy ];
```
</Tab>

<Tab>
Download the pre-built binary for your platform.

**Quick install script (coming soon):**
```bash
# This feature will be available in a future release
# curl -fsSL https://install.compozy.com | sh
```

**Manual download:**
Download the appropriate binary for your OS and architecture from our [GitHub Releases](https://github.com/compozy/compozy/releases) page.

1. Download the archive for your platform (e.g., `compozy_x.x.x_linux_x86_64.tar.gz`)
2. Extract the archive: `tar -xzf compozy_*.tar.gz`
3. Move the binary to your PATH: `sudo mv compozy /usr/local/bin/`
4. Make it executable: `sudo chmod +x /usr/local/bin/compozy`
</Tab>

</Tabs>

Verify the installation:

```bash
compozy --version
```

## Initialize Your First Project

Create a new Compozy project:

```bash
# Initialize a new project
compozy init my-ai-project
cd my-ai-project

# Or with Docker infrastructure setup
compozy init my-ai-project --docker
```

## Configuration

### Environment Variables

Create a `.env` file in your project:

```bash title=".env"
# AI Provider (choose one)
OPENAI_API_KEY=sk-your-openai-key
GROQ_API_KEY=gsk_your-groq-key
ANTHROPIC_API_KEY=sk-ant-your-anthropic-key

# Infrastructure (adjust for your setup)
DATABASE_URL=postgresql://user:pass@localhost:5432/compozy
REDIS_URL=redis://localhost:6379
TEMPORAL_ENDPOINT=localhost:7233
```

### Start Development

```bash
# Start your development server
compozy dev
```

After this you will have your server running at `http://localhost:5001`.

## Quick Verification

Test your setup with a simple workflow:

```bash
# List available workflows
compozy workflow list

# Execute the default greeter workflow
compozy workflow execute greeter --json='{"name": "World", "style": "friendly"}'
```
