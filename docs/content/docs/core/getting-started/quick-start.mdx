## Prerequisites

Ensure you have these installed:
- [Docker and Docker Compose](https://docs.docker.com/get-docker/) - For running services
- [Bun](https://bun.sh/docs/installation) - As runtime for your workflow
- [Compozy CLI](/docs/core/getting-started/installation) - For initializing and managing your project

<Steps numbered size="sm">
<Step title="Initialize a new project" description="Run compozy init to generate a new project">

```bash
compozy init my-weather-app
cd my-weather-app
```

</Step>

<Step title="Configure your project" description="Update `compozy.yaml` to your liking. Here you will add your workflows, models and other global configurations.">

```yaml title="compozy.yaml"
name: my-weather-app
version: "0.1.0"
description: A simple weather advisory system

workflows:
  - source: ./workflow.yaml

models:
  - provider: openai
    model: gpt-4o-mini
    api_key: "{{ .env.OPENAI_API_KEY }}"

runtime:
  type: bun
  entrypoint: "./main.ts"
  permissions:
    - --allow-read
    - --allow-net
    - --allow-env
```

</Step>
<Step title="Environment Variables" description="Create a `.env` file with your API keys">

```bash title=".env"
OPENAI_API_KEY=your_openai_api_key_here
```

</Step>

<Step title="Defining your tools" description="Let's now create a tool that will get weather data for a city">

```typescript title="weather_tool.ts"
interface WeatherInput {
  city: string;
}

interface WeatherOutput {
  temperature: number;
  humidity: number;
  weather: string;
  city: string;
}

export function weatherTool({ input }: { input: WeatherInput }): WeatherOutput {
  // TODO: Implement your tool logic here
  return {
    temperature: 22,
    humidity: 65,
    weather: "sunny",
    city: input.city,
  }
}
```

Then create the runtime entrypoint `main.ts`:

```typescript title="main.ts"
import { weatherTool } from "./weather_tool.ts";

export default {
  "weather_tool": weatherTool,
}
```

<Callout type="warning">
You need to export your tool using the `tool_id` as the key here, if you don't, the tool will not be available in your workflow.
</Callout>

</Step>

<Step title="Creating your first workflow" description="Workflows are the top-level orchestration units that define a sequence of tasks to accomplish a goal">

```yaml title="workflow.yaml"
id: weather-advisor
version: 0.1.0
description: Get weather info and activity recommendations

schemas:
  - id: city_input
    type: object
    properties:
      city:
        type: string
        description: The city to get weather for
    required:
      - city

config:
  input:
    $ref: local::schemas.#(id=="city_input")

tools:
  - id: weather_tool
    description: Get current weather for a city
    input:
      $ref: local::schemas.#(id=="city_input")

agents:
  - id: advisor
    config:
      # Uses OpenAI from compozy.yaml
      $ref: global::models.#(provider=="openai")
    instructions: |
      You are a helpful weather advisor.
      Based on weather data, suggest:
      1. Appropriate clothing
      2. Outdoor activities
      3. Any weather-related tips
      Respond in a friendly, conversational tone.
    tools:
      - $ref: local::tools.#(id=="weather_tool")
    actions:
      - id: get_weather
        prompt: |
          What is the current weather in {{ .workflow.input.city }}?
      - id: give_advice
        prompt: |
          Based on the weather in {{ .workflow.input.city }}:
          {{ .tasks.get_weather.output | toJson }}
          Please provide helpful advice for today.

tasks:
  - id: get_weather
    type: basic
    $use: agent(local::agents.#(id=="advisor"))
    action: get_weather
    with:
      city: "{{ .workflow.input.city }}"
    on_success:
      next: get_advice

  - id: get_advice
    type: basic
    $use: agent(local::agents.#(id=="advisor"))
    action: give_advice
    final: true

outputs:
  city: "{{ .workflow.input.city }}"
  weather: "{{ .tasks.get_weather.output }}"
  advice: "{{ .tasks.get_advice.output }}"
```

</Step>

<Step title="Testing your workflow" description="Compozy provides a local development server that you can use to test your workflow">

```bash
# From your project directory
compozy dev
```

After running your have a local server running at `http://localhost:3001`.

<Tabs items={["Rest Client", "cURL"]}>

<Tab>
You can use any HTTP client to test your workflow. We high recommend using [Rest Client](https://marketplace.visualstudio.com/items?itemName=humao.rest-client) extension for VSCode because it provides an easy way to test your workflow.

```http title="test.http"
@baseUrl = http://localhost:3001/api/v0
@workflowId = weather-advisor

### Execute the workflow
POST {{baseUrl}}/workflows/{{workflowId}}/execute
Content-Type: application/json

{
  "input": {
    "city": "San Francisco"
  }
}

### Check execution status (replace {execution_id} with actual ID)
@execId = {{exec.response.body.data.exec_id}}
GET {{baseUrl}}/executions/{{execId}}
```

</Tab>

<Tab>

Or you can use cURL to test your workflow.

```bash
# Execute the workflow
curl -X POST http://localhost:8000/api/v0/workflows/weather-advisor/execute \
  -H "Content-Type: application/json" \
  -d '{"input": {"city": "San Francisco"}}'
```

</Tab>

## Behind the Scenes

Here's what actually happens when you execute a Compozy workflow:

<Mermaid chart={`sequenceDiagram
    participant Client
    participant API as HTTP API
    participant Worker as Worker Service
    participant Temporal
    participant Executor as Task Executor
    participant Runtime as Bun Runtime
    participant LLM as OpenAI
    participant DB as PostgreSQL

    Client->>API: POST /workflows/weather-advisor/execute
    API->>Worker: TriggerWorkflow(input: {city: "SF"})
    Worker->>Worker: Validate input & Generate exec ID
    Worker->>Temporal: StartWorkflow(CompozyWorkflow)

    Note over Temporal: Durable workflow orchestration
    Temporal->>Executor: Execute Activity: TriggerWorkflow
    Executor->>DB: Create workflow state (status: running)

    Note over Temporal,Runtime: Task 1: get_weather (Basic Task + Agent)
    Temporal->>Executor: Execute Activity: Task[get_weather]
    Executor->>DB: Create task state
    Executor->>LLM: GenerateContent(action: get_weather)
    LLM->>LLM: Process with tool definitions
    LLM->>Runtime: Execute weather_tool({city: "SF"})
    Runtime-->>LLM: {temp: 22, weather: "sunny"}
    LLM-->>Executor: Complete with tool result
    Executor->>DB: Update task state (status: completed)

    Note over Temporal,LLM: Task 2: get_advice (Basic Task + Agent)
    Temporal->>Executor: Execute Activity: Task[get_advice]
    Executor->>DB: Create task state
    Executor->>LLM: GenerateContent(action: give_advice)
    LLM-->>Executor: Generated advice text
    Executor->>DB: Update task state (status: completed)

    Note over Temporal: Workflow completion
    Temporal->>Executor: Execute Activity: CompleteWorkflow
    Executor->>DB: Update workflow state (status: completed)
    Temporal-->>Worker: Workflow result
    Worker-->>API: Execution result
    API-->>Client: {exec_id, status, outputs}`} />

<List>
  <ListItem title="HTTP API Layer">
    The WorkflowController receives your request at `/api/v0/workflows/{id}/execute`
  </ListItem>
  <ListItem title="Worker Service Validation">
    Validates input against JSON schema, generates unique execution ID, and loads workflow configuration
  </ListItem>
  <ListItem title="Temporal Workflow Engine">
    Provides durable execution that survives crashes, manages retry logic with exponential backoff, and records complete execution history
  </ListItem>
  <ListItem title="Task Execution Loop">
    Each task runs as a Temporal activity with built-in retry policies and transactional state persistence
  </ListItem>
  <ListItem title="Agent Intelligence">
    LLM receives custom instructions and tool definitions, autonomously decides when to call tools, with execution in sandboxed Deno/Bun runtime
  </ListItem>
  <ListItem title="State Management">
    PostgreSQL stores workflow and task states with optimistic locking and complete audit trail
  </ListItem>
</List>
