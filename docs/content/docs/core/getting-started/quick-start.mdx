## Prerequisites

Ensure you have these installed:
- [Docker and Docker Compose](https://docs.docker.com/get-docker/) - For running services
- [Bun](https://bun.sh/docs/installation) - As runtime for your workflow
- [Compozy CLI](/docs/core/getting-started/installation) - For initializing and managing your project

<Steps numbered size="sm">
<Step title="Initialize a new project" description="Run compozy init to generate a new project">

```bash
compozy init hello-world
cd hello-world
```

</Step>

<Step title="Configure your project" description="Update `compozy.yaml` with basic configuration">

```yaml title="compozy.yaml"
name: hello-world
version: "0.1.0"
description: A simple greeting workflow

workflows:
  - source: ./workflows/main.yaml

models:
  - provider: openai
    model: gpt-4.1-2025-04-14
    api_key: "{{ .env.OPENAI_API_KEY }}"

runtime:
  type: bun
  entrypoint: "./entrypoint.ts"
  permissions:
    - --allow-read
    - --allow-net
    - --allow-write

autoload:
  enabled: true
  strict: true
  include:
    - "agents/*.yaml"
    - "tools/*.yaml"
  exclude:
    - "**/*~"
    - "**/*.bak"
    - "**/*.tmp"
```

</Step>

<Step title="Environment Variables" description="Create a `.env` file with your API key">

```bash title=".env"
OPENAI_API_KEY=your_openai_api_key_here
```

</Step>

<Step title="Create a simple tool" description="Create a tool that generates personalized greetings">

```typescript title="greeting_tool.ts"
interface GreetingInput {
  name: string;
  style?: string;
}

interface GreetingOutput {
  message: string;
  timestamp: string;
}

export async function greetingTool({ input }: { input: GreetingInput }): Promise<GreetingOutput> {
  const { name, style = "friendly" } = input;
  const styles = {
    friendly: `Hello there, ${name}! Hope you're having a great day!`,
    formal: `Good day, ${name}. It's a pleasure to meet you.`,
    casual: `Hey ${name}! What's up?`,
    enthusiastic: `${name}! So excited to meet you! ðŸŽ‰`
  };

  const message = styles[style as keyof typeof styles] || styles.friendly;
  return {
    message,
    timestamp: new Date().toISOString()
  };
}
```

Create the runtime entrypoint:

```typescript title="entrypoint.ts"
import { greetingTool } from "./greeting_tool.ts";

export default {
  "greeting_tool": greetingTool,
}
```

</Step>

<Step title="Create your workflow" description="Define a simple workflow that generates personalized greetings">

```yaml title="workflows/main.yaml"
id: hello-world
version: 0.1.0
description: Generate personalized greetings

schemas:
  - id: greeting_input
    type: object
    properties:
      name:
        type: string
        description: The person's name to greet
      style:
        type: string
        description: Greeting style (friendly, formal, casual, enthusiastic)
        default: friendly
    required:
      - name
      - style

config:
  input:
    $ref: schemas.#(id=="greeting_input")

tools:
  - id: greeting_tool
    description: Generate personalized greetings
    input:
      $ref: schemas.#(id=="greeting_input")

agents:
  - id: greeter
    config:
      $ref: global::models.#(provider=="openai")
    instructions: |
      You are a friendly assistant that creates personalized greetings.
      Use the greeting tool to generate an appropriate greeting, then enhance it with a thoughtful message.
    tools:
      - $ref: local::tools.#(id=="greeting_tool")
    actions:
      - id: create_greeting
        prompt: |
          Please create a greeting for {{ .workflow.input.name }} using the {{ .workflow.input.style }} style.
          After generating the greeting, add a brief, personalized message to make it more engaging.

tasks:
  - id: greet
    type: basic
    $use: agent(local::agents.#(id=="greeter"))
    action: create_greeting
    final: true

outputs:
  greeting: "{{ .tasks.greet.output }}"
  user: "{{ .workflow.input.name }}"
  style: "{{ .workflow.input.style }}"
```

</Step>

<Step title="Test your workflow" description="Start the development server and test your workflow">

```bash
# Start the development server
compozy dev
```

Your server will be running at `http://localhost:5001`.

## Behind the Scenes

Here's what happens when you execute this simple workflow:

<Mermaid chart={`sequenceDiagram
    participant Client
    participant API as HTTP API
    participant Worker as Worker Service
    participant Temporal
    participant Executor as Task Executor
    participant Runtime as Bun Runtime
    participant LLM as GPT-4.1
    participant DB as PostgreSQL

    Client->>API: POST /workflows/main-workflow/executions
    API->>Worker: TriggerWorkflow(input: {name: "Alice", style: "friendly"})
    Worker->>Worker: Validate input & Generate exec ID
    Worker->>Temporal: StartWorkflow(CompozyWorkflow)

    Note over Temporal: Durable workflow orchestration
    Temporal->>Executor: Execute Activity: TriggerWorkflow
    Executor->>DB: Create workflow state (status: running)

    Note over Temporal,Runtime: Task: greet (Basic Task + Agent)
    Temporal->>Executor: Execute Activity: Task[greet]
    Executor->>DB: Create task state
    Executor->>LLM: GenerateContent(action: create_greeting)
    LLM->>LLM: Process with tool definitions
    LLM->>Runtime: Execute greeting_tool({name: "Alice", style: "friendly"})
    Runtime-->>LLM: {message: "Hello there, Alice! Hope you're having a great day!", timestamp: "2024-01-01T12:00:00Z"}
    LLM-->>Executor: Complete with enhanced greeting
    Executor->>DB: Update task state (status: completed)

    Note over Temporal: Workflow completion
    Temporal->>Executor: Execute Activity: CompleteWorkflow
    Executor->>DB: Update workflow state (status: completed)
    Temporal-->>Worker: Workflow result
    Worker-->>API: Execution result
    API-->>Client: {exec_id, status, outputs}`} />

<List className="mt-12">
  <ListItem title="HTTP API Layer">
    The WorkflowController receives your request at `/api/v0/workflows/main-workflow/executions`
  </ListItem>
  <ListItem title="Input Validation">
    The system validates that the required `name` field is provided and the optional `style` field is valid
  </ListItem>
  <ListItem title="Temporal Workflow Engine">
    Provides durable execution that survives crashes and manages the workflow lifecycle
  </ListItem>
  <ListItem title="Task Execution">
    The single `greet` task runs with the greeter agent, which uses the greeting tool to generate a response
  </ListItem>
  <ListItem title="Agent Intelligence">
    GPT-4.1 receives the greeting instructions and autonomously calls the greeting tool, then enhances the response
  </ListItem>
  <ListItem title="Tool Execution">
    The greeting tool runs in the sandboxed Bun runtime and returns a structured response
  </ListItem>
</List>
