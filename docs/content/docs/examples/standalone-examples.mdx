## Overview

This page links to five complete, runnable examples that demonstrate how to use Compozy in standalone mode: basic local development, persistence, mixed deployments, edge constraints, and migration to distributed.

## Examples

- Basic Standalone — Minimal setup for local dev
  - Path: `examples/standalone/basic`
  - Workflow: `workflows/hello-world.yaml`
  - Docker: PostgreSQL only

- With Persistence — Embedded Redis snapshots (BadgerDB)
  - Path: `examples/standalone/with-persistence`
  - Workflow: `workflows/stateful-workflow.yaml`
  - Docker: PostgreSQL

- Mixed Mode — Standalone cache + external Temporal
  - Path: `examples/standalone/mixed-mode`
  - Workflow: `workflows/distributed-workflow.yaml`
  - Docker: Temporal + PostgreSQL

- Edge Deployment — Resource-constrained environments
  - Path: `examples/standalone/edge-deployment`
  - Workflow: `workflows/edge-workflow.yaml`
  - Docker: optional, includes `Dockerfile.edge`

- Migration Demo — Standalone → Distributed
  - Path: `examples/standalone/migration-demo`
  - Phases: `phase1-standalone`, `phase2-distributed`
  - Workflow: `workflows/migration-workflow.yaml`

## Prerequisites

- Go 1.25+
- Docker (Compose) for services where applicable

## Quick Start

From any example directory:
```bash
docker compose up -d  # if the example includes compose
go run ../../../main.go --config.from=env
```

Then execute a workflow using the REST API:
```bash
curl -sS -X POST 'http://localhost:5001/api/v0/workflows/<workflow_id>/executions' \
  -H 'Content-Type: application/json' \
  -d '{"input":{}}'
```

<Callout type="info">Tests run each example workflow end-to-end using the mock model to ensure stability and speed.</Callout>
