## Shared Decisions
- The checked-in `web/dist` contract is a tracked `.keep` placeholder restored after `web` builds so fresh checkouts and later embed work can rely on the directory existing without committing generated assets.
- Daemon web UI work must consume the additive `compozy-daemon` contract/client surface instead of replacing the existing `agh` OpenAPI/client path, because the current AGH app still depends on the legacy browser contract.
- Browser-facing memory/document transport must stay on daemon-issued typed payloads: the query layer exposes memory files via opaque `file_id` values and typed missing/stale-document errors instead of relative paths.
- Browser-scoped daemon HTTP requests now use `X-Compozy-Workspace-ID` as the canonical active-workspace input; shared handlers still fall back to legacy query/body workspace values so UDS and older daemon clients keep working, and missing or stale browser workspace context returns `412` in the shared `TransportError` envelope.
- Browser hardening is isolated to the HTTP transport: Host/Origin validation and double-submit CSRF are enforced only on the localhost HTTP listener, while UDS remains the higher-trust transport and intentionally does not share browser cookies or CSRF state.
- The canonical run SSE contract is `run.snapshot`, `run.event`, `run.heartbeat`, and `run.overflow` with `Last-Event-ID` / cursor resume semantics; client decoders retain legacy heartbeat/overflow compatibility so older watchers do not silently break during the transition.

## Shared Learnings
- Root `bun run build` and `bun run typecheck` now discover the new workspaces cleanly while the existing SDK workspaces continue to build/typecheck through the same Turbo graph.
- `bun run codegen` / `bun run codegen-check` are wired at the repo root via `scripts/codegen.mjs` (thin shell over `openapi-typescript`). `web/package.json` proxies through `bun run --cwd .. codegen[-check]` and also runs `web/scripts/tsr-generate.mjs` so `web/src/routeTree.gen.ts` is regenerated before typecheck/tests. The earlier shared-memory note that both AGH and `compozy-daemon` contracts already had codegen in this worktree was incorrect — only the `compozy-daemon` flow exists here; no AGH browser contract runs in this repo.
- oxfmt reformats every `.ts` file unless excluded, which rewrites the generated OpenAPI output and breaks `codegen-check`. The `.prettierignore` file at the repo root now pins `web/src/generated/**` and `web/src/routeTree.gen.ts` out of the format pass — keep those entries in sync if additional generated targets show up.
- `openapi-fetch` (v0.17) does not parse body fields out of a top-level `headers` option when the OpenAPI spec declares a typed header parameter. All workspace-scoped requests must pass the workspace header through `params: { header: { "X-Compozy-Workspace-ID": id } }`. Use the `ACTIVE_WORKSPACE_HEADER` constant exported from `@/systems/app-shell`.
- `web/src/styles.css` now imports `@compozy/ui/tokens.css` and declares `@source "../../packages/ui/src"`, which is the current contract for letting Tailwind scan shared package classes from the SPA workspace.
- The mockup display/mono assets are now owned by `packages/ui/src/assets/fonts/`, so later tasks should extend the shared package instead of pointing runtime CSS back at `docs/design/daemon-mockup/`.
- `~/.compozy/runs/{hooks-integration,run-job-hooks,review-hooks,artifact-hooks}/run.db` gets left behind by `internal/core/extension/hooks_integration_test.go` with fixed RunIDs; stale copies from a newer binary can hit `rundb: schema too new`. Remove those four dirs if the extension tests fail with that error — do not treat it as a regression in the current change.
