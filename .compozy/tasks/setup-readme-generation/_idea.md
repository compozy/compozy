# Setup README Generation

## Overview

Add a static `.compozy/README.md` generation step to the `compozy setup` command. After bundled skills are installed, the command writes a quick-start reference document inside the `.compozy/` directory. This file serves as an in-project cheat-sheet for the developer who configured Compozy, listing available commands with copy-paste examples so they can start workflows immediately.

## Problem

Developers who run `compozy setup` get skills installed but receive no persistent reference about what to do next. The `.compozy/` directory exists in their project but contains no documentation explaining its purpose or the available workflows. Today, the only way to recall available commands is `compozy --help` — which requires switching to a terminal and knowing the binary name.

### Market Data

CLI tooling has converged on generating documentation artifacts during setup. Tools like ReadmeGen, InitGen, and autoreadme-cli all produce README files as part of their scaffold flow. The AI coding assistant ecosystem (Claude Code's `/init` + CLAUDE.md, Microsoft AgentRC) generates context files during setup as standard practice. Compozy already generates agent context files (AGENTS.md, skill files) but lacks a human-facing quick-start reference in the workflow directory.

## Core Features

| #   | Feature                  | Priority | Description                                                                                                                                                                             |
| --- | ------------------------ | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| F1  | Static README generation | Critical | Write a `.compozy/README.md` file with auto-generated header, project description, and command listing after successful `compozy setup`                                                 |
| F2  | Command cheat-sheet      | Critical | List every major workflow command (`setup`, `start`, `exec`, `fetch-reviews`, `fix-reviews`, `sync`, `archive`, `validate-tasks`, `migrate`, `upgrade`) with one-liner usage examples    |
| F3  | Auto-generated header    | High     | Include `<!-- This file is auto-generated by compozy setup. Do not edit. -->` to warn users about overwrite behavior                                                                    |
| F4  | Silent overwrite         | High     | Always regenerate the README on subsequent `setup` runs without prompting                                                                                                               |
| F5  | Contract test            | High     | Test that asserts registered Cobra command names appear in the README content to prevent drift                                                                                           |

## Council Insights

- **Recommended approach:** Static embedded README with auto-generated header, command cheat-sheet format, and contract test for drift prevention.
- **Key trade-offs:** Static content simplicity vs. dynamic accuracy. Resolved by contract test that catches drift at test time.
- **Risks identified:** (1) README drift from CLI changes — mitigated by contract test. (2) User customization loss on overwrite — mitigated by auto-generated header warning.
- **Stretch goal (V2+):** Dynamic Workflow Launcher README that includes project state (active PRDs, pending tasks, review rounds).

## Out of Scope (V1)

- **Dynamic content based on installed skills/agents** — adds template complexity; defer to V2
- **Directory structure documentation** — explaining `.compozy/tasks/`, `config.toml`, etc. is a separate concern
- **User-customizable README template** — the file is auto-generated, not user-authored
- **Interactive `compozy quickstart` command** — complementary feature, different implementation path
- **Localization / multi-language README** — English only for V1
