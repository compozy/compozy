# ADR-001: Static README Generation in Setup Command

## Status

Accepted

## Date

2026-04-10

## Context

The `compozy setup` command installs bundled skills into agent directories but does not create any documentation artifact in the `.compozy/` directory. Users who run setup lack a quick-reference for available workflows and commands. The feature should serve as a functional quick-start guide without adding complexity to the setup flow.

## Decision

Generate a static `.compozy/README.md` file at the end of a successful `compozy setup` execution. The file will contain an auto-generated header, a listing of available commands with one-liner examples, and will be silently overwritten on subsequent setup runs. A contract test will assert that registered Cobra command names appear in the README content to prevent drift.

## Alternatives Considered

### Alternative 1: Dynamic README with installed skill/agent context

- **Description**: Generate a README that reflects which skills were installed, which agents were detected, and includes dynamic paths.
- **Pros**: More accurate to the user's actual setup state.
- **Cons**: Adds template rendering complexity, requires passing install state to the generator, harder to test.
- **Why rejected**: Overengineered for V1; user requested MVP with static content.

### Alternative 2: No README, rely on CLI help

- **Description**: Users use `compozy --help` and subcommand help instead of a file.
- **Pros**: Zero implementation cost, always accurate.
- **Cons**: Not discoverable when browsing `.compozy/` directory; does not serve as a quick-start guide.
- **Why rejected**: Does not address the user's need for a reference document inside the project directory.

## Consequences

### Positive

- Low implementation cost (single embedded string + write call)
- Immediate onboarding value for users after setup
- Contract test catches command drift automatically

### Negative

- Static content may lag behind CLI changes between test runs
- Silent overwrite loses any user customizations (mitigated by auto-generated header warning)

### Risks

- README content drift: mitigated by contract test asserting command names
- User confusion about overwrite: mitigated by auto-generated header

## Implementation Notes

- Hook into `executeInstall` in `internal/cli/setup.go` after successful skill installation
- Use `model.CompozyDir()` for the target path
- Include `<!-- This file is auto-generated by compozy setup. Do not edit. -->` header
- Contract test in `internal/cli/setup_test.go` or `internal/setup/` test files
