# BUG-001: `run.post_start` observers can trail `job.pre_execute`

**Severity:** High
**Priority:** P1
**Type:** Functional
**Status:** Fixed

## Environment

- **Build:** `e7eb8a8`
- **OS:** `darwin arm64`
- **Browser:** n/a
- **URL:** n/a

## Summary

The baseline `make verify` run exposed a daemon/extension lifecycle ordering bug: under full-suite load, extension observers sometimes saw `job.pre_execute` before `run.post_start`. That made the extension hook contract load-dependent and caused `internal/core/extension.TestHookDispatchIntegrationAcrossRunAndJobPhases` to fail intermittently during the repository gate.

## Reproduction

```bash
make verify
```

Observed before the fix:

- `internal/core/extension.TestHookDispatchIntegrationAcrossRunAndJobPhases` failed with `hook-100 hook order mismatch: expected "run.post_start" before "job.pre_execute"`.

## Expected

`run.post_start` observer delivery should complete before the executor begins `job.pre_execute`, so extensions always observe a stable run lifecycle ordering.

## Root cause

`emitRunStart` submitted the `run.started` event and queued `run.post_start` through the asynchronous observer dispatcher, but it returned immediately. The first job then entered synchronous `job.pre_execute` before the observer queue drained, so hook order became scheduler-dependent under load.

## Fix

- Added a shared executor helper that waits for pending observer hooks with a bounded timeout.
- Called that helper immediately after dispatching `run.post_start`, so execution cannot advance into job hooks before observer delivery completes.
- Added focused regression coverage in `internal/core/run/executor/execution_test.go` to lock the contract.

## Verification

- `go test ./internal/core/run/executor -run 'Test(EmitRunStartWaitsForObserverHooksBeforeReturning|FinalizeExecutionWaitsForObserverHooksWithoutCanceledRunContext)' -count=1`
- `go test -race -parallel=4 ./internal/core/extension -count=1`
- `make verify`

## Impact

- **Users Affected:** extension observers attached to daemon-managed task/review/exec runs
- **Frequency:** sometimes, especially under heavier suite load
- **Workaround:** none
