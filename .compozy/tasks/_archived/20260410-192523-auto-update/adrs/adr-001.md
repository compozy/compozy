# ADR-001: Use creativeprojects/go-selfupdate for CLI Auto-Update

## Status

Accepted

## Date

2026-04-06

## Context

Compozy CLI is distributed via GoReleaser as Homebrew cask, NPM package, `go install`, and direct binary download. Users need a way to update the CLI (`compozy upgrade`) and be notified when a new version is available. The update mechanism must integrate with GoReleaser's archive naming, `checksums.txt` (SHA256), and cosign signatures.

Three libraries were evaluated: `creativeprojects/go-selfupdate`, `minio/selfupdate`, and `rhysd/go-github-selfupdate`.

## Decision

Use `creativeprojects/go-selfupdate` as the primary library for release detection, download, checksum validation, and binary replacement.

The library provides:
- Native GoReleaser integration (`ChecksumValidator` with `UniqueFilename: "checksums.txt"`)
- GitHub Releases API source with semantic version parsing
- Archive format support (tar.gz, zip) matching GoReleaser output
- ARM architecture detection matching GoReleaser's naming
- Context-based API with cancellation support
- Atomic binary replacement with rollback on failure

## Alternatives Considered

### Alternative 1: Minimal stdlib net/http + minio/selfupdate

- **Description**: Use stdlib HTTP to query GitHub Releases API, parse JSON manually, compare versions with a semver library, then use `minio/selfupdate` only for binary replacement.
- **Pros**: Full control, no magic, minimal dependencies
- **Cons**: Significant boilerplate for archive naming, checksum validation, platform detection. Must reimplement GoReleaser naming conventions.
- **Why rejected**: Too much custom code to maintain for a solved problem. GoReleaser naming conventions are non-trivial.

### Alternative 2: rhysd/go-github-selfupdate

- **Description**: Older library, predecessor to `creativeprojects/go-selfupdate`.
- **Pros**: Battle-tested, simpler API
- **Cons**: No context.Context support, no GitLab/HTTP sources, less actively maintained, no GoReleaser-specific features
- **Why rejected**: Superseded by `creativeprojects/go-selfupdate` which was forked from it with significant improvements.

## Consequences

### Positive

- Single dependency handles the full update lifecycle (detect, download, validate, replace)
- GoReleaser integration is automatic — no custom naming/checksum logic needed
- Future GitLab or private repository support available without code changes

### Negative

- Adds a non-trivial transitive dependency tree (HTTP client, archive handling, crypto)
- Library updates may require testing against GoReleaser output changes

### Risks

- Library maintenance risk: mitigated by the library being actively maintained and widely used
- Breaking changes in GitHub API: mitigated by the library abstracting the API interaction

## Implementation Notes

Configure the updater with `ChecksumValidator{UniqueFilename: "checksums.txt"}` to match the existing `.goreleaser.yml` checksum configuration. Use `selfupdate.ParseSlug("compozy/compozy")` as the repository source.
