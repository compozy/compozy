# Compozy Web Dashboard — Consolidated Audit (2026-04-23)

**Scope.** Full audit of the React 19 + TanStack Router + Tailwind 4 + shadcn SPA
at `web/`, served by the Go daemon on port `2323` via `web/embed.go` and the
handlers under `internal/api/httpapi/**` + `internal/api/core/**`. Audit covers
visual design, UX, realtime data pipeline, and technical architecture.

The audit was produced by five parallel analysis passes (mapping, UI,
UX/features, realtime, architecture) and consolidated here. The full per-pass
reports are in the session transcript.

---

## 1. Executive Summary

The dashboard has solid bones:

- Type-safe API via `openapi-typescript` + `openapi-fetch`.
- Clean `systems/{feature}` folder convention.
- SSE with cursor resume, heartbeat, and overflow for runs
  (`web/src/systems/runs/lib/stream.ts`, `use-run-stream.ts`).
- Storybook-as-test + MSW handler-group composition.
- CSRF double-submit + host/origin allowlist middleware on the Go side.

But the **product is visibly below par** for an operator console, and three
failures dominate:

1. **The SSE stream is thrown away.** `RunDetailView` receives every event the
   journal emits (jobs, tool calls, content blocks, usage, heartbeats) and
   uses them only to increment an "events received" counter. The TUI streams
   live logs; the web shows a static snapshot until you reload.
2. **The UI vocabulary is one-note.** Every page is `SectionHeading → maybe an
   inline alert → a stack of identical `SurfaceCard`s`. No data tables, no
   tabs, no tooltips, no toasts, no skeletons, no command palette. 15+
   copy-pasted alert divs. Markdown rendered as `<pre>`.
3. **The Go static server ships without security or caching hardening.** No
   CSP, no X-Frame-Options, no immutable Cache-Control on hashed assets, no
   gzip/brotli, an SPA fallback that 200s on any dotless path.

Fixing items 1, 2, and 3 unlocks the rest of the roadmap and removes the "this
feels like a prototype" perception users hit on minute one.

## 3. Implementation Plan

The order optimises for user-visible impact per hour. The live event feed (C1)
changes the product's character — it is the single highest-leverage fix.

### Phase 1 — Foundation (CRITICAL tokens + primitives)

1. Fix `packages/ui/src/tokens.css`: drop `--spacing` override, raise
   `--card` contrast, set `--ring` to brand primary, unify brand greens.
2. Add `Alert`, `Skeleton`, `Metric` primitives to `packages/ui`.
3. Mount `<Toaster />` (from `sonner`) in `web/src/main.tsx`.

### Phase 2 — Live runs (CRITICAL realtime)

4. Add `web/src/systems/runs/lib/event-store.ts` — bounded ring buffer +
   `useSyncExternalStore` subscription.
5. Add `web/src/systems/runs/components/run-event-feed.tsx` — renders
   events with per-kind icons + timestamps.
6. Wire `onEvent` in `runs_.$runId.tsx` → append to store; render the feed
   inside `RunDetailView`.
7. Ensure `runKeys.list()` invalidates on terminal events so list reflects
   finished runs.

### Phase 3 — Content polish

8. Add `@compozy/ui` `Markdown` component (`react-markdown` + `rehype-sanitize`
   + `remark-gfm`) and swap `<pre>` usages in reviews/spec/memory/task views.
9. Replace inline alert divs across views with `<Alert>`.
10. Replace text loading states with `<Skeleton>` row patterns.
11. Dashboard: replace active-run scalar with live active-runs list; add
    `refetchInterval: 3000` to dashboard + runs list queries.
12. Runs list: add duration column + a workflow filter.

### Phase 4 — Go hardening

13. Add security headers middleware in `internal/api/httpapi/server.go`.
14. Patch `internal/api/httpapi/static.go` to serve hashed assets with
    `Cache-Control: public, max-age=31536000, immutable` and `index.html`
    with `no-cache`, plus strong ETag.

### Phase 5 — Post-verify follow-ups (documented, not shipped this pass)

- Daemon `POST /api/runs/{id}/retry` + web wire-up.
- Command palette (`cmdk`) + keyboard shortcuts.
- Route loaders + `queryOptions` factories + Suspense.
- Review/task SSE endpoints + generic `useEventStream<T>` in
  `web/src/systems/realtime/`.
- Data-table primitive and migration of runs/reviews lists.
- Parametrised run launcher modal (TUI `validation_form.go` equivalent).
- Memory search / edit / delete endpoints.
- CSP tightening once Tailwind nonces/hashes are wired.

This pass implements Phases 1–4 and documents Phase 5 as the next slice.
